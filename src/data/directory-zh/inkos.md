InkoS（Ink Operating System）由开发者Narcooo于2026年发布，是一款针对长篇小说的多智能体自动写作系统。为了解决常见的角色遗忘、伏笔断裂、逻辑混乱及词汇重复等长文本生成难题，InkoS采用独特的“真相文件（Truth Files）”架构，将写作过程拆分为写作（Writer）、审计（Auditor）和修订（Revisor）三个智能体阶段，实时同步并更新七类剧情知识和世界观状态。系统不仅能实现从零开始的全自动流水线创作，还支持导入现有未完结小说文本，逆向解析并重建世界观，实现无缝续写。

凭借终端灵活设计，InkoS提供全自动模式、基于JSON命令的原子操作模式及自然语言交互Agent模式，是面向专业文字创作者与AI开发者的高质量长文本写作工业基础设施。

## 功能列表

- **真相文件（Truth Files）底层架构**：为每本书动态维护七套独立真相文件，精确追踪角色信息、资源消耗、世界观限制和词汇频次，根治多章节长篇AI写作中的幻觉问题。
- **逆向工程及续写能力**：支持导入本地小说文件或章节目录，结合正则表达式`--split`自定义分章，自动拆解旧作并构建角色矩阵与伏笔状态，实现连续无缝续写。
- **全自动多智能体闭环创作**：分工明确的写作管线，起草智能体专注描写，审计智能体依照多项检查标准校验剧情，修订智能体根据审计报告进行文本修正。
- **人工审查关卡机制**：支持设置多章节批量连续写作，自动挂起任务引入人工监督，确保故事主线符合创作者意图。
- **三大工作模式支持**：包括“一键完整管线”生成，支持细粒度JSON命令接口，以及包含13个工具的自然语言Agent会话。
- **自动词汇疲劳检测**：内置词汇监控器，有效避免高频重复及陈词滥调，提升文本的真实感与丰富度。

## 使用帮助

### 1. 环境准备与安装

InkoS为TypeScript实现的CLI程序，使用前请确保具备合适的开发环境。

- **环境配置**：请访问Node.js官网下载安装最新LTS版本（推荐18.x及以上）。
- **代码下载与安装**：终端中运行如下命令：

```bash
git clone https://github.com/Narcooo/inkos.git
cd inkos
npm install
npm run build
npm link
```

- **设置API密钥**：请将您的OpenAI API密钥设置为环境变量：

```bash
export OPENAI_API_KEY="您的_API_KEY_填写于此"
```

### 2. 三种核心工作流模式

InkoS支持三种操作模式，适合不同程度的创作者需求。

#### 模式一：完整管线模式（一键全自动生产）

这是高效产出的模式，自动完成写作、审计和修订流程。

- **单章节生成与润色**

```bash
inkos write next 吞天魔帝
```

  系统自动写作下一章节草稿，执行多维度逻辑审计，最终由修订模块完成定稿。

- **批量连续写作**

```bash
inkos write next 吞天魔帝 --count 5
```

  可连续自动创作多章节，无需人工干预，完成后自动保存。

#### 模式二：原子命令模式（精细控制与集成）

为高级用户设计，可分解处理流程并自定义操作。

- **限定大纲草拟**

```bash
inkos draft 吞天魔帝 --context "本章重点写师徒矛盾爆发，主角愤而离家出走" --json
```

  通过`--context`强制指导剧情，`--json`输出结构化数据，便于结合如[OpenClaw](https://www.kdjingpai.com/openclaw-ai/)等外部Agent自动化。

- **单独调用审计**

```bash
inkos audit 吞天魔帝 31 --json
```

  对自写稿件进行AI审计，找出逻辑漏洞。

- **自动修订**

```bash
inkos revise 吞天魔帝 31 --json
```

  根据审计结果自动修订文本。

#### 模式三：自然语言智能体（Agent）模式

无需记忆指令，使用自然语言与系统对话指挥创作。

- **场景对话下达命令**

```bash
inkos agent "帮我写一本都市修仙类型的小说，主角是个程序员，马上帮我建档并写好第一章。"
```

- **综合策略安排**

```bash
inkos agent "先扫描目前的网文市场趋势，然后根据热门数据为我创建一本新书的大纲。"
```

### 3. 高级功能：导入旧作与无缝续写

支持导入旧文本，利用逆向工程功能，恢复世界观并实现自动续写。

- **步骤A：准备文本文件**

  将已写章节保存为本地文本文件。

- **步骤B：导入并拆解文本**

```bash
inkos import 吞天魔帝 --file ./novel.txt --split "第[一二三四五六七八九十百千万\d]+章"
```

  `--split`支持正则表达式定义章节分割，如途中断电，可用`--resume-from <章节号>`断点续写。

- **步骤C：重建世界观与真相文件**

  系统自动解析文本，生成7份真相文件，涵盖角色状态、伏笔和世界约束。

- **步骤D：自动续写**

```bash
inkos write next 吞天魔帝
```

  系统继承原文风及伏笔，顺畅延续故事。

## 应用场景

1. 提升网络长篇小说的更新频率和产能，通过自动化流水线减轻写作压力，确保剧情逻辑严密。
2. 修复老坑或进行同人续写，系统能完美复刻旧作风格和伏笔，实现无缝接续。
3. 管理复杂游戏剧情和动态设定，精准追踪人物与资源状态变化，适用于大型叙事游戏和桌游。
4. 为开发者提供底层命令接口，方便集成至各类智能体和自动化创作平台，构建无人值守内容生产体系。

## 常见问答

1. **如何避免AI忘记设定或虚构道具？**

   InkoS依赖“真相文件”工程架构，创建7个独立真值文件作为剧情唯一客观依据，从根本阻断长篇幻觉产生。

2. **能否单独使用审计功能？**

   完全可以，使用`inkos audit <书名> <章节>`指令调用审计智能体，为自写稿件逻辑把关。

3. **小说排版混乱怎么办？**

   支持灵活的章节切分正则表达式，针对特殊格式文本准确分章，且支持断点续导，保证导入海量旧稿稳定。
